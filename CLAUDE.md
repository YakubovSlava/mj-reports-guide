# CLAUDE.md — Руководство по созданию отчётов для платформы MJ

## Что такое пользовательские отчёты

Платформа MJ поддерживает навык **«Пользовательские отчёты»**: администратор загружает Python-скрипт, пользователи запускают его из браузера и видят HTML-результат прямо на странице.

Ты помогаешь написать такой скрипт по описанию задачи.

---

## Контракт скрипта

```python
import sys

data_path = sys.argv[1] if len(sys.argv) > 1 else None
# data_path — путь к CSV-файлу (или None, если источник не задан)

# ... обработка данных ...

print("""
<div>HTML-отчёт здесь</div>
""")
```

**Правила:**
- Скрипт получает путь к данным через `sys.argv[1]` (может отсутствовать)
- Весь вывод — `print(html_string)`, никакого Flask/HTTP
- Таймаут выполнения — 60 секунд
- Доступны только стандартные библиотеки Python + то, что установлено в venv проекта: `pandas`, `openpyxl`
- Не пиши в файлы, не делай сетевые запросы

## Форматы данных

### funnel_data.csv
```
report_dt, dash_channel, dash_product, sort, step_name, value
```
- `report_dt` — дата (YYYY-MM-DD)
- `dash_channel` — канал (PUSH, Email, SMS, …)
- `dash_product` — продукт (Дебетовая карта, Кредит, …)
- `sort` — порядковый номер шага воронки
- `step_name` — название шага
- `value` — количество

### complaints.csv
```
report_dt, toxic_flag, unit, subj, prd, sum_cnt
```
- `report_dt` — дата
- `toxic_flag` — тип (Жалобы / Обращения)
- `unit` — подразделение (Розничный блок, …)
- `subj` — тема жалобы
- `prd` — продукт
- `sum_cnt` — количество

### client_metrics.csv
```
id, unit, product_name, date_, fvalue, step_name, step_id, metodika, step_codename
```
- `unit` — бизнес-юнит
- `product_name` — продукт
- `date_` — дата (ISO)
- `fvalue` — значение метрики
- `step_name` — название показателя

### clickstream_monthly.csv / clickstream_weekly.csv
```
report_dt, channel, product, sessions, clicks, conversions, revenue
```

### dzo_report.xlsx
Excel-файл с несколькими листами. Читать через `openpyxl` или `pandas`.

## Стили платформы

Платформа подключает `/static/style.css`. Используй переменные CSS — они автоматически адаптируются под тёмную тему:

```css
var(--bg)          /* фон страницы */
var(--surface)     /* фон карточек */
var(--border)      /* граница */
var(--text)        /* основной текст */
var(--text-muted)  /* второстепенный текст */
var(--accent)      /* акцентный цвет (синий) */
```

Готовые классы:
- `.summary-card` — карточка с метрикой
- `.funnel-table` — таблица с конверсионными шагами
- `.conv-high` / `.conv-mid` / `.conv-low` — бейдж конверсии (≥60% / ≥30% / <30%)

## Как писать хороший скрипт

1. **Всегда проверяй** `data_path is None` — выводи понятное сообщение об ошибке
2. **Кодировка**: открывай CSV с `encoding='utf-8-sig'` (есть BOM в некоторых файлах)
3. **Разделитель**: пробуй `;` и `,`, или используй `pandas` с `sep=None, engine='python'`
4. **Экранируй HTML**: если вставляешь пользовательские строки в HTML, используй `html.escape()`
5. **Форматируй числа**: большие числа через `f"{x:,.0f}".replace(",", " ")`
6. **Не используй внешние CSS-фреймворки** — только inline-стили и переменные платформы

## Шаблон ответа Claude

Когда пользователь описывает задачу, ты:
1. Уточняешь источник данных (какой CSV-файл)
2. Уточняешь нужные метрики / фильтры
3. Предлагаешь структуру отчёта (что покажем)
4. Пишешь скрипт целиком — готовый к загрузке
5. Описываешь, как его загрузить: Админка → вкладка «Отчёты» → кнопка «+ Добавить отчёт»
